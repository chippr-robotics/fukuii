# 0.7.0 — SNAP healing at mainnet scale, concurrent sync pipelines, eth/69

Promotion of `staging` (0.6.x series) to `main`. Highlights since 0.6.0:

## Sync & state healing
- **BFS frontier traversal** with a RocksDB-backed level queue and batched
  `multiGet` node reads — full-state healing walks at ETC-mainnet scale
  (~119M trie nodes) run in hours instead of days (#1323, #1325).
- **Healing frontier scalability**: bounded LRU visited set (L1) and
  persisted frontier with completeness-marker resume (L2) (#1318), with
  single-flight walk gating, pivot-refresh deferral, and walk-failure
  recovery (#1325).
- **ConcurrentFetch pipelines** for fast- and regular-sync header/body/
  receipt fetching with peer rate tracking (#1323).
- StateHeal verification-on-pivot-refresh + dead-pulse watchdog (#1321).

## Networking
- **eth/69 support hardening**: PoW chain-weight calibration without a TD
  field (#1323) and a tolerant 3-arm `Status69` decode that keeps
  non-canonical peers from churning the peer pool (#1324).
- External IP auto-detect, Mordor DNS discovery, fork-PR-friendly CI
  (#1321, #1322).

## Operations
- Grafana SNAP-sync and node-health dashboards aligned with the BFS
  healing pipeline; dormant recovery panels removed (#1325).
- Parameterized container heap/memory via environment (#1323).

Validated end-to-end on the barad-dûr ETC-mainnet node (85 GB state):
single-flight BFS walk, flat memory at 30M+ entry levels, eth/69 peer
pool stability, and Mordor regular-sync catch-up at ~130 blocks/s.

Released from the `staging` promotion of 2026-06-11.
